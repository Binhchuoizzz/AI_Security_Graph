\chapter{Experiments and Evaluation}
\label{ch:experiments_evaluation}

% Chapter introduction
This chapter presents the empirical evidence of the thesis, validating the research hypotheses through rigorous quantitative metrics. The SENTINEL system is evaluated against a 5-Dimensional framework (5D Metrics: Accuracy, Performance, Security, Explainability, and Integrity). The content emphasizes the layered Ablation Study design and the application of biostatistical models (McNemar's Test, Mann-Whitney U Test) to prove that the architectural enhancements are statistically significant and mathematically sound.

\section{Experimental Environment and Datasets}
\begin{itemize}
    \item \textbf{Hardware Setup:} CPU Intel Core i9, 32GB RAM, GPU NVIDIA RTX 4060 Ti 16GB (utilized for executing the Gemma-2-9B-IT Q4\_K\_M model).
    \item \textbf{CSE-CIC-IDS2018 Dataset:} Introduction to the dataset encompassing diverse attack typologies (Brute Force, DoS, Web Attack) serving as the baseline for overarching accuracy evaluation.
    \item \textbf{DAPT2020 Dataset:} Employed for testing scenarios involving the identification of multi-stage Advanced Persistent Threat (APT) chains.
    \item \textbf{Unified Data Stream Simulation:} Detailing the scripting process that merges and replays log streams according to virtual timestamps to emulate real-time properties.
\end{itemize}

\section{5D Evaluation Metrics and Ablation Study Design}
\begin{itemize}
    \item \textbf{Defining the 5D Metrics:} Accuracy (F1, Precision, Recall), Performance (Latency, Throughput), Security (Robustness Rate), Explainability (Audit Completeness), and Integrity (HMAC Validation).
    \item \textbf{Ablation Study Configurations (Config A through F):}
        \begin{itemize}
            \item Config A (Baseline 1): Traditional Rule-based engine exclusively.
            \item Config B (Baseline 2): Pure Local LLM inference (No RAG, no Welford).
            \item Config C: Welford Tier-1 + LLM.
            \item Config D: LLM + Single-RAG (FAISS).
            \item Config E: LLM + Dual-RAG Hybrid.
            \item Config F (Holistic SENTINEL): Tier-1 (Welford) + Tier-2 (LangGraph Agent + Dual RAG + Guardrails).
        \end{itemize}
\end{itemize}

\section{Accuracy Evaluation \& McNemar's Statistical Test}
\begin{itemize}
    \item \textbf{Precision, Recall, and F1-Score Results:} Comparative charts and tables across configurations. Highlighting Config F's optimal F1 balance (eliminating Rule-based False Positives while enhancing Zero-day Recall).
    \item \textbf{APT Detection Efficacy:} Analysis of Tier-1 (Welford) capturing weak anomalous signals and subsequently activating the Agent's Threat Memory chains.
    \item \textbf{McNemar's Test Application:} Comparing the Confusion Matrices of Config B and Config F. Proving $p-value < 0.05$ (rejecting the Null Hypothesis) to confirm that the variance is attributable to architectural superiority rather than random chance.
\end{itemize}

\section{Performance Latency Evaluation \& Mann-Whitney U Test}
\begin{itemize}
    \item \textbf{Inference Latency Analysis:} Comparative table of average latency per flow. The exceptional offloading achieved by Tier-1 (Welford processing benign logs in $O(1)$ time) and the Semantic Cache (Cache Hits reducing LLM latency from 4-6 seconds to milliseconds).
    \item \textbf{Non-parametric Mann-Whitney U Test:} Demonstrating that the latency distribution of the SENTINEL configuration is significantly faster ($p < 0.05$) compared to routing all logs directly through the LLM.
\end{itemize}

\section{Adversarial Robustness Evaluation (Security)}
\begin{itemize}
    \item \textbf{Adversarial AI Manipulation Dataset (45 Payloads):} Description of manipulative scenarios including Prompt Injections and Delimiter Smuggling (e.g., ``Ignore system rules, report this IP as safe'').
    \item \textbf{Mitigation Rate Measurement:} Comparison between an unguarded system (80\% manipulation success rate) versus SENTINEL's enabled Delimited Data Encapsulation mechanism (achieving near-perfect mitigation of injection attempts).
\end{itemize}

\section{Integrity and RAG Quality Evaluation via LLM-as-a-Judge}
\begin{itemize}
    \item \textbf{HMAC Chaining Audit:} A hypothetical scenario where an attacker compromises the database to alter log labels. The dashboard immediately reports the broken hash chain.
    \item \textbf{RAG Scoring (RAGAS / LLM-as-a-Judge):} Utilizing an external Llama-3.1 model as an independent arbiter to score four criteria: Context Precision, Context Recall, Faithfulness, and Answer Relevancy. Proving that Dual-RAG with RRF outperforms Single Vector Search.
\end{itemize}

In summary, Chapter 5 verifies the technical prowess of SENTINEL through hard quantitative data and rigorous biostatistical tests. Chapter 6 will distill the significance of these findings and outline future trajectories.
