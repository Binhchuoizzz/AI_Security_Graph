\chapter{The Proposed SENTINEL Architecture}
\label{ch:proposed_solution}

% Chapter introduction
This chapter meticulously delineates the design of the SENTINEL system (Cognitive Two-Tier Architecture). Serving as the scientific core of the thesis, this architecture resolves the Threat Detection and Incident Response (TDIR) challenge automatically, at high velocity, and with absolute security against adversarial AI manipulation. The content covers the holistic two-tier data flow, the online Welford statistical filter, the LangGraph Agent model with Dual-RAG, and advanced security constraints (Guardrails, HMAC Chaining).

\section{Overview of the Two-Tier Cognitive Architecture}
\begin{itemize}
    \item \textbf{Holistic System Block Diagram:} Illustrating the data flow starting from log generation, traversing Tier-1 (Transport/Filter Layer), escalating to Tier-2 (Cognitive/AI Agent Layer), and culminating in the Database/Dashboard.
    \item \textbf{Design Philosophy:} Separation of concerns—Tier-1 is strictly responsible for wire-speed noise filtration, while Tier-2 provides deep contextual analysis to resolve the blindness of static systems toward Zero-day threats.
\end{itemize}

\section{Tier-1: Real-Time Traffic Filtration Engine (RuleEngine \& Welford)}
\begin{itemize}
    \item \textbf{Session and Baseline Management:} Mechanisms for storing and updating network connection states temporally.
    \item \textbf{Online Welford's Algorithm Mechanism:} Mathematical specification for computing Z-Scores per log stream. When a Z-Score exceeds threshold $\alpha$, the system generates an ESCALATE label for Tier-2 instead of dropping the packet.
    \item \textbf{Static Rule Engine Integration:} Incorporation of fundamental signature-based rules (DROP, WARN) to instantly eliminate rudimentary, known attack patterns.
\end{itemize}

\section{Tier-2: LangGraph Cognitive AI Agent}
\begin{itemize}
    \item \textbf{StateGraph Design:} The architectural structure of the \texttt{SentinelState} object carrying session information. Definition of core Nodes: Analyze, Retrieve, Resolve.
    \item \textbf{Conditional Edges Routing:} The internal reasoning loop of the AI Agent. Upon identifying a knowledge deficit, the agent autonomously transitions to the \texttt{retrieve\_knowledge} state before returning to evaluation.
\end{itemize}

\section{Dual-RAG Mechanism and Semantic Memory}
\begin{itemize}
    \item \textbf{Dual-RAG Architecture:} Unification of two specialized knowledge bases: MITRE ATT\&CK (TTP identification) and NIST SP 800-61r2 (Incident Response Handbook) \cite{nist80061}.
    \item \textbf{Hybrid Search Algorithm:} The fusion of Dense (FAISS) and Sparse (BM25) scoring mechanisms utilizing RRF.
    \item \textbf{Semantic Cache:} A semantic hashing mechanism to bypass LLM inference if a highly similar log stream was successfully analyzed earlier in the temporal window, drastically reducing latency.
\end{itemize}

\section{Threat Memory and APT Campaign Correlation}
\begin{itemize}
    \item \textbf{Graph Data / IP Correlation Storage (SQLite/Neo4j):} Logic for tracking the Reputation Score of IP addresses over prolonged timelines.
    \item \textbf{APT Correlation Mechanism:} An algorithm to chain temporally scattered, low-severity anomalies into a cohesive high-severity escalation verdict, exposing low-and-slow campaigns.
\end{itemize}

\section{AI Security Guardrails Layer}
\begin{itemize}
    \item \textbf{Delimited Data Encapsulation:} An algorithm generating a cryptographically random token per session (e.g., \texttt{<|PAYLOAD\_A7F9|>}) to encapsulate foreign data, ensuring the LLM does not execute hidden adversarial instructions.
    \item \textbf{Token Budget Manager \& LogTemplateMiner:} Utilizing Drain3 to strip random textual variations and cluster logs, thereby preventing LLM Token Denial-of-Service (DoS) attacks.
    \item \textbf{Output Sanitization Filter:} Strict standardization and JSON schema enforcement of the LLM's output.
\end{itemize}

\section{Data Integrity and HMAC Protection}
\begin{itemize}
    \item \textbf{HMAC Log Chaining:} A cryptographic hashing algorithm linking consecutive logs, ensuring that any tampering by malicious insiders breaks the chain and is immediately flagged.
    \item \textbf{RAG SHA-256 Checksum Validation:} Guaranteeing the vector database is not compromised by injected falsified documents (Data Poisoning).
\end{itemize}

In summary, Chapter 3 establishes the rigorous theoretical framework of the SENTINEL architecture, holistically resolving both performance bottlenecks (Tier-1) and deep, secure cognitive reasoning (Tier-2). Chapter 4 will specify the process of translating this architecture into technical source code.
