Triết lý Thiết kế Tầng 1 (Lọc Thô Chi Phí Thấp)

Đảm nhận vai trò "Tấm lưới chắn tốc độ cao", xử lý 100% lưu lượng mạng ở tốc độ đường truyền mà không đụng tới GPU, loại bỏ ngay các truy cập lành tính hoặc tấn công dạng chữ ký thô.

Triết lý Thiết kế Tầng 2 (Bộ Não Thám Tử)

Dành riêng sức mạnh suy luận của LLM cho thiểu số sự kiện thực sự mơ hồ hoặc nghi vấn zero-day, được bảo vệ bằng rào chắn mật mã và đối chiếu tri thức MITRE ATT&CK.

Chi tiết Kỹ thuật — Tier 1

Bộ Lọc Tốc Độ Đường Truyền Phi Trạng Thái (RuleEngine & Welford)

Thuật Toán Trực Tuyến Welford $O(1)$

Tính toán trung bình $\mu_k$ và phương sai lăn $\sigma_k^2$ liên tục trong bộ nhớ mà không cần lưu lại lịch sử gói tin:

Z = (x_k - \mu_k) / \sigma_k # Ngưỡng kích hoạt: Z > 3.5σ -> ESCALATE TO ML / TIER 2
  • Cập nhật Khóa-theo-Phán-quyết: Chỉ cập nhật baseline với luồng benign, chống đầu độc "ếch luộc" (slow-boil poisoning).
  • Golden Baseline Seed: Khởi tạo trạng thái sạch từ trước, tránh hiện tượng rò rỉ khi khởi động lạnh.
Tối Ưu Hóa Bộ Nhớ & Tra Cứu Cổng
  • Tra cứu Cổng $O(1)$ Set: Thay vì chạy Regex đắt đỏ, RuleEngine dùng cấu trúc Set $O(1)$ để lọc nhanh các port nhạy cảm.
  • Bộ Đệm Bounded `_rep_cache`: Giới hạn dung lượng cache uy tín IP trong bộ nhớ, chống tràn RAM khi bị DDoS tấn công diện rộng.
  • Session Baseline TTL: Bảng băm tự dọn dẹp các session hết hạn, giữ chân RAM luôn ở ngưỡng an toàn.

Chi tiết Kỹ thuật — Tier 1.75

Semantic Cache: Bỏ Qua LLM Cho Các Tấn Công Lặp Lai

Triết Lý Bỏ Qua Độ Trễ LLM

Khi bị tấn công DDoS hoặc Brute-Force, hàng nghìn log có cấu trúc tương tự nhau đổ về. Gọi LLM cho từng log lặp lại là lãng phí tài nguyên GPU nghiêm trọng.

Payload -> Drain Parser -> SHA-256 Hash -> Cache Lookup (1ms)
Thông Số Vận Hành Semantic Cache
  • Thuật toán Khai thác Drain: Rút gọn log thô thành mẫu chuẩn (template), loại bỏ các tham số ngẫu nhiên.
  • Cấu trúc LRU Cache: Lưu trữ trong bộ nhớ (`OrderedDict`) tối đa 500 mục với TTL 1800 giây.
  • Hiệu quả Thực tế: Giảm độ trễ phán quyết từ 4–6 giây xuống ~1 mili-giây cho 100% các mẫu đã nhận diện.
Chi tiết Kỹ thuật — Tier 2

Tác Tử Nhận Thức LangGraph & Local LLM (Gemma-2-9B)

Đồ Thị Trạng Thái FSM LangGraph

Chạy mô hình `Gemma-2-9B-IT` cục bộ qua `llama.cpp` server (cửa sổ 8,192 tokens, temp=0.1):

  • Pydantic Schema Decoding: Ép LLM xuất định dạng `LLMDecision` nghiêm ngặt.
  • Fault Tolerance (`_salvage_fields`): Dự phòng bằng Regex cứu dữ liệu nếu JSON bị lỗi, tự về `AWAIT_HITL` nếu thất bại.
Tối Ưu Hóa Prompt KV-Caching

Để giảm thời gian tạo token đầu tiên (TTFT):

  • Static System Prompt: Giữ nguyên cố định System Prompt chứa hướng dẫn an ninh để `llama.cpp` cache KV.
  • Volatile User Feedback: Chuyển toàn bộ các thông tin thay đổi vào User Message, giúp tăng tốc độ phản hồi đáng kể.
Tri thức & Trạng thái

Cơ Chế Dual-RAG & Bộ Nhớ Đe Dọa SQLite

Dual-RAG: Kết Hợp Dày & Thưa
Phương pháp Thuật toán Điểm mạnh
Dày (Dense) FAISS + MiniLM Hiểu ngữ nghĩa, từ đồng nghĩa
Thưa (Sparse) BM25 Score Chính xác ID, CVE, Cổng, IP

$\rightarrow$ Dung hợp kết quả bằng Reciprocal Rank Fusion (RRF $k=60$).

Bộ Nhớ Đe Dọa & Liên Kết APT
  • Cấu hình SQLite: `synchronous=NORMAL` + Indexes. TUYỆT ĐỐI KHÔNG DÙNG WAL để tránh lỗi crash phân quyền cross-UID trong Docker.
  • Suy Giảm Uy Tín IP: Uy tín IP suy giảm theo thời gian $S_{new} = S_{old} \times (1-\lambda)^t$.
  • Block-on-Sight: IP có uy tín = 100 bị Tầng 1 chặn ngay lập tức mà không cần gọi LLM.
  • Hardcoded Whitelist: Các IP nội bộ hạ tầng (vd `10.0.0.99`) tuyệt đối không bị chặn.
Bảo mật AI & Toàn vẹn

Rào Chắn Bảo Mật AI & Đóng Gói Dữ Liệu Phân Định

Đóng Gói Dữ Liệu Phân Định (Nonce)

Chống lại tấn công Prompt Injection từ Log Payload thô:

<<>> [UNTRUSTED USER PAYLOAD HERE] <<>>

Mọi token lệnh bên trong ranh giới Nonce mật mã đều bị ép coi là dữ liệu thụ động, từ chối quyền thi hành lệnh đối kháng.

Chuỗi HMAC Audit & Guardrails
  • Móc Xích Log HMAC-SHA256: Mỗi log được băm liên hoàn $H_i = \text{HMAC}(D_i \parallel H_{i-1}, K)$, phát hiện ngay lập tức nếu kẻ tấn công sửa xóa log.
  • Consensus Guardrail: Tự động ghi đè LLM về `AWAIT_HITL` nếu phán quyết hạ cấp mâu thuẫn tín hiệu rủi ro Tầng 1.
  • T1571 Guardrail: Các tấn công cổng chung chung (MITRE T1571) bắt buộc ép về `AWAIT_HITL` để tránh chặn nhầm.

  • Thực tiễn: Sản phẩm mã nguồn mở đóng gói Docker Compose hoàn chỉnh, chạy air-gapped thành công trên GPU phổ thông (RTX 4060 Ti).
Lộ Trình Tiến Lên Production
  • Viết lại Tầng 1 bằng Go/Rust: Loại bỏ overhead Python để xử lý hàng triệu events/giây.
  • Continuous Batching (vLLM / TGI): Thay `llama.cpp` bằng vLLM để phán xử hàng chục sự kiện mỗi chu kỳ GPU.
  • Message Broker Kafka: Thay thế Redis Streams bằng Kafka đĩa cứng cho doanh nghiệp lớn.
