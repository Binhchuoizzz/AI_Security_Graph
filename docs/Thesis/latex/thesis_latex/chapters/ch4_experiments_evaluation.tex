\section{Security and Lessons Learned}
\textbf{1. Prompt Injection Resistance:}
The static guardrail layer wrapping the Payload in nonce hashes enabled SENTINEL to block $100\%$ of obfuscated attacks (Base64/Hex). Combined with the consensus guardrail, the system thwarted all attempts to hijack the LLM (Table~\ref{tab:robustness}).

\begin{table}[!htbp]
    \centering
    \caption{Static guardrail layer blocking rates on 120 payloads. (Source: Author)}
    \label{tab:robustness}
    \begin{tabular}{|l|c|}
        \hline
        \textbf{Attack Group} & \textbf{Block Rate} \\ \hline
        Encoding Bypass (Base64/Hex/URL) & 100.0\% \\ \hline
        Structural / Delimiter Attacks & 35.0\% \\ \hline
        \textbf{Overall (Static)} & \textbf{50.0\%} \\ \hline
    \end{tabular}
\end{table}

\textbf{2. Fixing Evidence Stripping:}
Initially, the system inadvertently stripped special characters from the Payload before feeding it to the LLM (mistaking them for delimiters). This blinded the LLM to SQLi/XSS attacks. \textbf{Solution:} Transmitting the raw Payload (including quotes and brackets) entirely and exclusively encapsulating it with normalized tags \texttt{<<<DATA\_BEGIN...>>>}. SQLi detection rate instantly surged to nearly 99\%.

\textbf{3. Fixing RAG Pollution:}
When raw, noisy Payloads were fed directly into FAISS, retrieval results were skewed. \textbf{Solution:} Forcing the ML Gateway to affix a preliminary MITRE label (e.g., T1190) and combining it with BM25 Keyword Search, resulting in a dramatic increase in retrieval precision.

\section{Summary}
Experimental results prove that SENTINEL's two-tier architecture offloads LLM processing by $83.8\%$, slashes latency by $82.97\%$ compared to pure LLM usage, and ensures absolute immunity to log injection attacks.
